# Pole Move Organizer

整理 pole dance 動作編碼清單：**不更動任何文字**，依數字由小到大排序。

## 規則

### 前綴處理
- 所有前綴（`c`、`s`、`~` 及其組合）**一律原樣保留，不得省略或修改**
- 例：`c326` → `c326`，`~c326` → `~c326`，`s335` → `s335`

### 排序
- 依每個 entry 的**數字部分**由小到大排序
- 數字相同的 entries 維持原有相對順序

### 保留
- 前綴、數字、數字後方的所有注記文字，全部原樣保留，一字不增刪
- 空格、括號、英文注記，全部照抄

## 輸入格式判斷

### 格式 A：逗號分隔（一般格式）
```
~c326, 328, 322 未串, s335, c360
```
→ 直接以逗號分隔解析

### 格式 B：緊湊格式（無逗號，c+數字連寫）
```
sc325c329c337一半c339前c405/
```
→ 在每個 `c+數字` 前切分；沒有數字的字母片段（如開頭的 `s`）合併到緊接的下一個 entry

判斷方式：輸入**不含逗號**且**中間有 `c` 後接數字**→ 視為格式 B，需先展開為逗號分隔再排序

## 執行步驟

1. 判斷輸入格式（A 或 B）
2. 格式 B：將緊湊串展開為各別 entry（見上方規則）
3. 對每個 entry：
   - 提取數字（用於排序，只讀不改）
   - entry 本身完整保留，不做任何修改
4. 依數字升冪排序
5. 以半形逗號 + 空格分隔輸出

## 輸出後驗證（必做）

排序完成後，**自動執行**以下檢查：
- 輸出的 entry 數量 = 輸入的 entry 數量
- 每個 entry 的文字內容與輸入完全一致（無增刪）
- 若有不一致，標示差異並告知用戶

## 範例

### 格式 A：逗號分隔

**輸入**：
```
~c326, 328, 329, 322 未串, 327一半 缺jallegra, 331, 334, 337,338 to ayesha, 339 (ball grip to chest), 340, 341, 343, 349, s335,c360, c368 學完未串,337, 371 honey split, 372, 373(~elbow),396
```

**輸出**：
```
322 未串, ~c326, 327一半 缺jallegra, 328, 329, 331, 334, s335, 337, 337, 338 to ayesha, 339 (ball grip to chest), 340, 341, 343, 349, c360, c368 學完未串, 371 honey split, 372, 373(~elbow), 396
```

**驗證**：輸入 22 個，輸出 22 個，一致。

### 格式 B：緊湊格式

**輸入**：
```
sc325c329c335c336c337一半c339前c405/
```

**展開**：`sc325`, `c329`, `c335`, `c336`, `c337一半`, `c339前`, `c405/`

**輸出**（已排序，本例原本即升序）：
```
sc325, c329, c335, c336, c337一半, c339前, c405/
```

**驗證**：7 個，一致。
